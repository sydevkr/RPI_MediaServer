## 시스템 접속

### 기본 정보
| 항목 | 값 |
|------|-----|
| RTSP URL | `rtsp://<라즈베리파이_IP>:8554/live` |
| 기본 포트 | 8554 |
| 스트림 경로 | `/live` |

### 접속 예시
```
rtsp://192.168.1.100:8554/live
```

## 영상 스트리밍 시청

### VLC 미디어 플레이어

**PC (Windows/Mac/Linux)**
1. VLC 실행
2. 미디어 → 네트워크 스트림 열기
3. URL 입력: `rtsp://<IP>:8554/live`
4. 재생 클릭

**명령줄**
```bash
vlc rtsp://192.168.1.100:8554/live
```

### 모바일 앱

**Android**
- [VLC for Android](https://play.google.com/store/apps/details?id=org.videolan.vlc)
- [RTSP Player](https://play.google.com/store/apps/details?id=net.butterflytv.rtspplayer)

**iOS**
- [VLC for Mobile](https://apps.apple.com/app/vlc-for-mobile/id650377962)
- [RTSP Viewer](https://apps.apple.com/app/rtsp-viewer/id1057427664)

### 웹 브라우저

**WebRTC (미지원)**
- 현재 RTSP 직접 재생은 브라우저에서 지원되지 않음
- HLS/HTTP-FLV 변환 필요 (추후 지원 예정)

### 보안 카메라 앱

**IP Cam Viewer**
- 프로토콜: RTSP
- 주소: `<IP>:8554`
- 경로: `/live`

**tinyCam Monitor**
- 카메라 유형: Generic RTSP
- RTSP URL: `rtsp://<IP>:8554/live`

## 뷰 모드 설명

시스템은 4가지 뷰 모드를 제공합니다. (운영자가 설정)

### Mode 1: HDMI 전용
```
┌─────────────────┐
│                 │
│  HDMI 영상      │
│  (1920x1080)    │
│                 │
└─────────────────┘
```

**특징**
- HDMI 입력 장치 단독 표시
- TV, 셋톱박스, 카메라 등 연결 시 사용

---

### Mode 2: 웹캠 전용
```
┌─────────────────┐
│                 │
│  웹캠 영상      │
│  (1920x1080)    │
│                 │
└─────────────────┘
```

**특징**
- USB 웹캠 단독 표시
- 화상회의, 감시용 웹캠 연결 시 사용

---

### Mode 3: Side-by-Side (좌우 분할)
```
┌────────┬────────┐
│        │        │
│ HDMI   │ Webcam │
│        │        │
└────────┴────────┘
```

**특징**
- 두 영상을 좌우로 동일 크기로 분할
- 각 영상은 960x1080로 스케일링
- 두 장소 동시 감시 시 유용

---

### Mode 4: PIP (Picture-in-Picture)
```
┌─────────────────┐
│  HDMI 영상      │
│                 │
│    ┌────────┐   │
│    │ Webcam │   │
│    │ (PIP)  │   │
│    └────────┘   │
└─────────────────┘
```

**특징**
- HDMI를 메인 영상으로 풀화면
- 웹캠을 우측 하단에 작게 (320x240) 오버레이
- 메인 화면 감시 + PIP로 자신 확인 시 유용

## AI 탐지 기능

### 탐지 개요
- **대상**: 사람 (Person)
- **하드웨어**: Hailo-8L AI 가속기
- **처리**: 실시간 (30fps 기준 약 25-30fps 유지)

### 화면 표시
```
┌─────────────────┐
│  ┌──────┐       │
│  │  👤  │ 85%   │  ← 바운딩 박스 + 신뢰도
│  └──────┘       │
│                 │
└─────────────────┘
```

**표시 요소**
- 녹색 사각형: 탐지된 객체 위치
- 퍼센트: 신뢰도 (0-100%)

### 알림 기능

**현재 지원**
- 실시간 화면 오버레이 표시

**추후 지원 예정**
- 모바일 푸시 알림
- 이메일 알림
- 녹화 자동 시작

## 자주 묻는 질문 (FAQ)

### Q: 접속이 안돼요
**A:** 다음을 확인하세요
1. 라즈베리파이와 동일 네트워크에 있는지 확인
2. `ping <라즈베리파이_IP>` 로 네트워크 연결 확인
3. 방화벽에서 포트 8554가 열려있는지 확인
4. MediaMTX 서버가 실행 중인지 확인

### Q: 영상이 끊겨요
**A:** 
- Wi-Fi 환경: 5GHz 대역 사용 권장
- 네트워크 대역폭: 최소 10Mbps 권장
- VLC 설정: 버퍼 크기 증가 (도구 → 환경설정 → 입력 및 코덱)

### Q: 화질이 안좋아요
**A:**
- 기본 설정: 1920x1080 @ 4Mbps
- 네트워크 상태에 따라 자동 조정될 수 있음
- 운영자에게 비트레이트 증가 요청

### Q: AI 탐지가 작동하지 않아요
**A:**
- Hailo-8L 하드웨어가 설치되어 있는지 확인
- 운영자에게 문의하여 로그 확인 요청
- 조명이 너무 어두우면 탐지가 어려울 수 있음

### Q: 모바일에서 소리가 안나요
**A:**
- RTSP 스트림은 기본적으로 비디오만 전송됨
- 오디오 포함 시 별도 설정 필요 (운영자 문의)

### Q: 녹화 기능이 있나요
**A:**
- 현재 실시간 스트리밍만 지원
- MediaMTX 설정으로 서버 측 녹화 가능 (운영자 설정 필요)
- VLC 등에서 클라이언트 측 녹화 가능

### Q: 여러 명이 동시에 볼 수 있나요
**A:**
- 네트워크 대역폭만 충분하다면 가능
- 각 클라이언트마다 별도 스트림 생성
- 3-4명까지 권장 (더 많을 경우 성능 저하)

### Q: RTSP 외 다른 프로토콜은?
**A:**
- 현재 RTSP만 지원
- HLS/HTTP-FLV 변환 지원 예정
- 웹 브라우저 재생을 위해서는 별도 미들웨어 필요
